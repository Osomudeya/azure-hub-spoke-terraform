4. **Communication (15%)**
   - Clear explanations
   - Professional presentation
   - Thoughtful answers

## Emergency Protocols

### If Live Demo Fails
1. Stay calm
2. Show screenshots immediately
3. Explain what should happen
4. Show your troubleshooting approach

### If Laptop Issues
1. Use their computer
2. Open your GitHub repo
3. Walk through the code
4. Use phone for screenshots

### If You Forget Something
- Say "Let me check my documentation"
- Pause and think
- Admit you don't know but explain how you'd find out
- NEVER make something up
